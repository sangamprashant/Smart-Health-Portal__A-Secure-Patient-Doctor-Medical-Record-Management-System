## What The MCA Guideline Requires

The guideline says the project report should:

- Be about `60 to 80 pages` excluding source code
- Be based on an actual `software development project`
- Show the full `SDLC`
- Include analysis, design, coding, testing, security, reports, and future scope
- Include both `PERT chart` and `Gantt chart`
- Include `SRS`
- Include suitable diagrams such as:
  - DFD
  - ER Diagram
  - UML Diagrams
  - State / Sequence / Use Case / Activity / Class diagrams as needed
- Include `data dictionary`
- Include `unit test cases` and `system test cases`
- Include `cost estimation`
- Include `bibliography`
- Include `glossary`
- Include `appendices`

## Formatting Rules From The Guideline

- Font: `Garamond`
- Body font size: `12`
- Main heading: `14 bold`
- Subheading: `12 bold italic`
- Line spacing: `1.5`
- Paper: `A4`
- Printing: `single side`
- Each main heading should start on a `new page`
- All pages, figures, and tables must be numbered
- Table titles and figure titles must be given
- Front and back cover: `Black` with `Golden` text
- The project title must remain exactly the same throughout the report

## 7. Problem Statement

Write clearly:

Healthcare systems often suffer from disconnected records, delayed emergency access, weak communication between patients and doctors, and lack of secure centralized data access. The proposed Smart Health Portal solves these issues by providing a secure web-based platform for record management, appointments, messaging, notifications, and emergency QR access.
